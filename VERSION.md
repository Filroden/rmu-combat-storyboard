# Version History

| Version | Changes |
| :--- | :--- |
| **Version 1.0.0** | **IMPROVEMENTS**<br>- Replaced the RMU size descriptor with a size calculated from the combatant's actual footprint so it accounts for any spells or effects that might increase or decrease the token's size.<br>- Determined the highest natural or worn AT on the torso to provide a simple description of the combatant's armour and also whether a shield is equipped.|
| **Version 1.0.0-beta1** | **Initial Beta Release**<br><br>**FEATURES**<br>- **Automated Combat Logging:** Hooks directly into RMU combats to silently record attacks, spells, resistance rolls, and applied damage.<br>- **Storyboard Wizard UI:** An interface accessed from the Combat Tracker to review and export the combat timeline.<br>- **Cast & Roster Management:** Allows the GM to replace actor names with physical descriptions to guide AI script and image generation.<br>- **Hero Moments:** Provides a toggle to spotlight specific combat actions so they become focal points in the generated script.<br>- **Prompt Export:** Compiles the curated combat data into a token-optimised prompt that instructs external AI models to generate both a narrative script and image generation prompts. <br>- **Plain Text Export:** Includes a one-click download to save a human-readable text file of the combat timeline.<br><br>**IMPROVEMENTS**<br>- *(Initial release - no prior improvements)*<br><br>**BUG FIXES**<br>- *(Initial release - no prior bug fixes)*<br><br>**UNDER THE HOOD**<br>- **Journal Data Storage:** Combat logs, custom rosters, and campaign context are saved as flags on standard Foundry Journal Entries so data persists between game sessions.<br>- **ETL Ingestion Pipeline:** Intercepts and parses hook data before it reaches the database so only relevant, sanitised data is stored.|
